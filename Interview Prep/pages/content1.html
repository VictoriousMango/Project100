Python Interview Preparation

Your one-stop guide to acing Python interviews

  1. What is Python? What are the benefits of using Python?
    Python is a high-level, general-purpose programming language known for its readability and versatility. **Benefits of using Python:** * **Readability:** Python's syntax is clean and easy to understand, making it beginner-friendly and reducing development time. * **Versatility:** Python can be used for various purposes, including web development, data science, machine learning, automation, and scripting. * **Large and Active Community:** A strong community provides extensive libraries, frameworks, and support resources. * **Cross-platform Compatibility:** Python code can run on different operating systems with minimal modifications. * **Extensive Libraries:** A vast collection of libraries (e.g., NumPy, Pandas, TensorFlow) simplifies complex tasks and accelerates development.
  2. What is a dynamically typed language?
    In a dynamically typed language, the data type of a variable is determined at runtime, not during compilation. This means you don't need to explicitly declare the data type of a variable before assigning a value to it.
  3. What is an Interpreted language?
    An interpreted language is executed line by line by an interpreter, unlike compiled languages that are translated into machine code before execution. This allows for more flexibility and easier debugging but can be slower than compiled languages.
  4. What is PEP 8 and why is it important?
    PEP 8 is a style guide for writing Python code. It provides recommendations for code formatting, naming conventions, and other stylistic aspects. **Importance of PEP 8:** * **Readability:** Following PEP 8 makes code more consistent and easier to read and understand, both for the original author and others. * **Maintainability:** Consistent code style improves code maintainability, making it easier to modify and debug in the future. * **Collaboration:** PEP 8 promotes a common coding style within a team or project, leading to better collaboration and code sharing.
  5. What is Scope in Python?
    Scope in Python refers to the region of the program where a variable is accessible. Different scopes include: * **Global Scope:** Variables defined outside any function or class are global and accessible from anywhere in the program. * **Local Scope:** Variables defined within a function are local and only accessible within that function. * **Enclosed Scope:** Variables defined in the enclosing function (if any) of a nested function. * **Built-in Scope:** Predefined variables like `len()`, `print()`, etc., are available in all scopes.
  6. What are lists and tuples? What is the key difference between the two?
    * **Lists:** Ordered, mutable (changeable) collections of elements enclosed in square brackets `[]`. * **Tuples:** Ordered, immutable (unchangeable) collections of elements enclosed in parentheses `()`. **Key Difference:** The primary difference is mutability. Lists can be modified by adding, removing, or changing elements, while tuples cannot be changed after creation.
  7. What are the common built-in data types in Python?
    Python offers several built-in data types: * **Integers (`int`)**: Whole numbers (e.g., 10, -5, 0) * **Floats (`float`)**: Numbers with decimal points (e.g., 3.14, -2.5) * **Strings (`str`)**: Sequences of characters (e.g., "Hello", 'world') * **Booleans (`bool`)**: Represents truth values (`True`, `False`) * **Lists (`list`)**: Ordered, mutable collections (e.g., [1, 2, 3], ["a", "b", "c"]) * **Tuples (`tuple`)**: Ordered, immutable collections (e.g., (1, 2, 3), ("a", "b", "c")) * **Sets (`set`)**: Unordered collections of unique elements (e.g., {1, 2, 3}, {"a", "b"}) * **Dictionaries (`dict`)**: Collections of key-value pairs (e.g., {"name": "Alice", "age": 30})
  8. What is pass in Python?
    `pass` is a placeholder statement in Python. It does nothing when executed. It's often used to fill in empty blocks of code temporarily to avoid syntax errors.
  9. What are modules and packages in Python?
    * **Modules:** A Python file (with the `.py` extension) containing Python code that can be imported and used in other Python scripts or modules. * **Packages:** A collection of related modules organized hierarchically within a directory. Packages allow for better organization and modularity of code.
  10. What are global, protected and private attributes in Python?
    * **Global Attributes:** Variables declared outside any class or function have global scope and can be accessed from anywhere in the program. * **Protected Attributes:** Attributes prefixed with a single underscore (`_`) are considered protected. They are intended for internal use within the class and its subclasses. While they can be accessed from outside the class, it is discouraged. * **Private Attributes:** Attributes prefixed with a double underscore (`__`) are considered private. They are not directly accessible from outside the class. However, they can be accessed indirectly using name mangling (e.g., `_ClassName__attributeName`).
  11. What is the use of self in Python?
    `self` is a convention used in Python's object-oriented programming. It refers to the instance of the class itself within a method. It's used to access the object's attributes and methods.
  12. What is __init__?
    `__init__` is a special method in Python classes called the constructor. It's automatically called when an object of the class is created. It's used to initialize the object's attributes with initial values.
  13. What are break, continue and pass in Python?
    * **break:** Immediately exits the current loop (for, while) and transfers control to the statement after the loop. * **continue:** Skips the current iteration of the loop and moves to the next iteration. * **pass:** A placeholder statement that does nothing. It's often used to maintain the correct syntax where a statement is required but no action is needed.
  14. What are unit tests in Python?
    Unit tests are small, isolated tests that verify the correctness of individual units of code (e.g., functions, methods, classes). They help ensure that each part of the code works as expected and improve code quality and maintainability.
  15. What is docstring in Python?
    A docstring is a string literal that appears as the first statement within a module, function, class, or method definition. It's used to document the purpose, usage, and parameters of the code element. Docstrings are accessible through the `__doc__` attribute.
  16. What is slicing in Python?
    Slicing is a mechanism used to extract a portion of a sequence (like lists, strings, tuples) using indices. It's done using the colon (`:`) operator. For example, `my_list[start:end:step]` extracts elements from `start` index to `end-1` index with a step size of `step`.
  17. Explain how can you make a Python Script executable on Unix?
    1. **Create a shebang:** Add `#!/usr/bin/env python3` (or the path to your Python interpreter) as the first line of your Python script. 2. **Make the script executable:** Use `chmod +x .py` in the terminal to make the script executable. 3. **Run the script:** Execute the script directly from the terminal using `./ .py`.
  18. What is the difference between Python Arrays and lists?
    * **Arrays:** - Provided by the `array` module. - Designed to store elements of the same data type (e.g., all integers, all floats). - Generally more memory-efficient for storing homogeneous data. * **Lists:** - More general-purpose. - Can store elements of different data types within a single list. - More flexible but may consume more memory compared to arrays for homogeneous data.
