Docker

  1. What is Docker

    Docker is a platform for developing, shipping, and running applications inside containers. Containers allow developers to package an application with its dependencies, ensuring consistency across different environments.

  2. Containers vs Virtual Machines

    Containers are lightweight, sharing the host OS kernel, while Virtual Machines (VMs) include a full guest OS, making them heavier but more isolated. Docker uses containers for efficiency.

    Aspect Containers Virtual Machines
    Size Small (MBs) Large (GBs)
    Startup Seconds Minutes
    OS Shares host kernel Full guest OS
  3. Docker Images

    A Docker image is a read-only template used to create containers. It includes the application, libraries, and dependencies. 

    docker pull python:3.9  # Pull an image from Docker Hub
  4. Docker Containers

    A container is a runnable instance of a Docker image. It isolates the application and its environment from the host system. 

    docker run -it python:3.9  # Run a container interactively
  5. Dockerfile

    A Dockerfile is a script containing instructions to build a Docker image, such as base image, dependencies, and commands. 

    FROM python:3.9
WORKDIR /app
COPY . .
RUN pip install flask
CMD ["python", "app.py"]
    docker build -t myapp .  # Build image from Dockerfile
  6. Docker Hub

    Docker Hub is a cloud-based registry for storing and sharing Docker images, both public and private. 

    docker push username/myapp:tag  # Push image to Docker Hub
  7. Docker Compose

    Docker Compose is a tool for defining and running multi-container Docker applications using a YAML file. 

    # docker-compose.yml
version: '3'
services:
  web:
    image: myapp
    ports:
      - "5000:5000"
  db:
    image: postgres
    environment:
      - POSTGRES_PASSWORD=example
    docker-compose up  # Start services
  8. Volumes

    Volumes persist data outside of containers, allowing data to survive container restarts or deletions. 

    docker run -v mydata:/data postgres  # Mount a volume
  9. Networking

    Docker provides networking options to allow containers to communicate with each other or the host, using bridge, host, or overlay networks. 

    docker network create my-network  # Create a custom network
docker run --network my-network myapp  # Run container on network
  1. Build and Run a Flask App

    Create a Dockerfile for a Flask app, build the image, and run it as a container. 

    # Dockerfile
FROM python:3.9
WORKDIR /app
COPY . .
RUN pip install flask
CMD ["python", "app.py"]
    docker build -t flask-app .
docker run -p 5000:5000 flask-app
    Output: Flask app running on port 5000
