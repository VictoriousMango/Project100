Multiprocessing and Multithreading in Python

  1. What is Concurrency

    Concurrency refers to the ability of a program to manage multiple tasks that can run independently or overlap in execution. In Python, concurrency can be achieved using Multithreading (for I/O-bound tasks) or Multiprocessing (for CPU-bound tasks).

  2. What is Multithreading

    Multithreading allows multiple threads to run within the same process, sharing the same memory space. It is ideal for I/O-bound tasks (e.g., file operations, network requests) but limited by Python's Global Interpreter Lock (GIL) for CPU-bound tasks. 

    import threading

def print_numbers():
    for i in range(5):
        print(f"Number: {i}")

thread = threading.Thread(target=print_numbers)
thread.start()
thread.join()
  3. What is Multiprocessing

    Multiprocessing creates separate processes, each with its own memory space and Python interpreter. It bypasses the GIL, making it suitable for CPU-bound tasks (e.g., heavy computations). 

    from multiprocessing import Process

def square_numbers():
    for i in range(5):
        print(f"Square: {i * i}")

process = Process(target=square_numbers)
process.start()
process.join()
  4. Global Interpreter Lock (GIL)

    The GIL is a mutex that protects access to Python objects, preventing multiple native threads from executing Python bytecodes at once. It simplifies memory management but limits true parallelism in multithreading for CPU-bound tasks.

  5. Thread Synchronization

    Synchronization mechanisms like Locks and Semaphores prevent race conditions when multiple threads access shared resources. 

    import threading

counter = 0
lock = threading.Lock()

def increment():
    global counter
    with lock:
        counter += 1

threads = [threading.Thread(target=increment) for _ in range(10)]
for t in threads:
    t.start()
for t in threads:
    t.join()
print(f"Counter: {counter}")

    from multiprocessing import Process, Queue

def producer(queue):
    queue.put("Hello from producer")

queue = Queue()
p = Process(target=producer, args=(queue,))
p.start()
p.join()
print(queue.get())
  7. Thread Pool and Process Pool

    Pools manage a fixed number of threads or processes to execute tasks concurrently, improving efficiency. 

    from multiprocessing import Pool

def square(n):
    return n * n

with Pool(4) as p:
    results = p.map(square, [1, 2, 3, 4])
print(results)
  8. Differences Between Multithreading and Multiprocessing

    Aspect Multithreading Multiprocessing
    Execution Unit Threads within a process Separate processes
    Memory Shared Separate
    GIL Impact Limited by GIL Bypasses GIL
    Use Case I/O-bound tasks CPU-bound tasks

  1. Parallel Sum Calculation

    Use multiprocessing to calculate the sum of a list in parallel. 

    from multiprocessing import Pool

def partial_sum(numbers):
    return sum(numbers)

numbers = [1, 2, 3, 4, 5, 6, 7, 8]
with Pool(2) as p:
    results = p.map(partial_sum, [numbers[:4], numbers[4:]])
total = sum(results)
print(total)
