Operating Systems

  1. What is an Operating System

    An Operating System (OS) is software that manages computer hardware, software resources, and provides services for programs. Examples include Windows, Linux, and macOS. It acts as an intermediary between users and hardware.

  2. Process Management

    The OS manages processes (running programs) by allocating CPU time, handling process states (e.g., running, waiting), and providing mechanisms for process creation and termination. 

    import os
print(f"Process ID: {os.getpid()}")  # Get current process ID
  3. Threads

    Threads are lightweight units of execution within a process. The OS schedules threads to utilize CPU resources efficiently. 

    import threading

def task():
    print(f"Thread ID: {threading.current_thread().ident}")

thread = threading.Thread(target=task)
thread.start()
thread.join()
  4. Memory Management

    The OS allocates and deallocates memory for processes, manages virtual memory, and ensures efficient use of RAM through techniques like paging and segmentation.

    (Demonstrated conceptually; Python abstracts memory management)

  5. File Systems

    The OS manages file storage, retrieval, and organization on disk using file systems (e.g., NTFS, ext4). It provides APIs for file operations. 

    import os
with open("example.txt", "w") as f:
    f.write("Hello, OS!")
print(os.path.exists("example.txt"))  # Check if file exists
  6. CPU Scheduling

    CPU scheduling determines which process or thread runs on the CPU at any given time. Common algorithms include First-Come-First-Serve (FCFS), Round Robin, and Priority Scheduling.

    (Managed by the OS; Python example simulates multitasking)

    from multiprocessing import Process

def task(name):
    print(f"Task {name} running")

processes = [Process(target=task, args=(i,)) for i in range(3)]
for p in processes:
    p.start()
for p in processes:
    p.join()
  7. Inter-Process Communication (IPC)

    IPC allows processes to communicate and synchronize. Methods include pipes, message queues, and shared memory. 

    from multiprocessing import Process, Pipe

def sender(conn):
    conn.send("Hello from sender")
    conn.close()

parent_conn, child_conn = Pipe()
p = Process(target=sender, args=(child_conn,))
p.start()
print(parent_conn.recv())
p.join()
  8. Deadlocks

    A deadlock occurs when two or more processes are unable to proceed because each is waiting for a resource the other holds. The OS detects and resolves deadlocks using algorithms like Banker’s Algorithm.

    (Conceptual; Python example simulates a simple deadlock)

    import threading

lock1 = threading.Lock()
lock2 = threading.Lock()

def thread1():
    with lock1:
        with lock2:
            print("Thread 1")

def thread2():
    with lock2:
        with lock1:
            print("Thread 2")

t1 = threading.Thread(target=thread1)
t2 = threading.Thread(target=thread2)
t1.start()
t2.start()
t1.join()
t2.join()
  9. Virtual Memory

    Virtual memory allows processes to use more memory than physically available by mapping virtual addresses to physical addresses, using paging or swapping.

    (Managed by the OS; no direct Python control)
