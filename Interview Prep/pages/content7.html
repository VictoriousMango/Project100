Git and GitHub

  1. What is Git

    Git is a distributed version control system that tracks changes in source code during software development. It allows multiple developers to collaborate on a project, manage versions, and revert changes if needed.

  2. What is GitHub

    GitHub is a platform built on top of Git that provides hosting for Git repositories, collaboration tools (e.g., pull requests, issues), and additional features like CI/CD integration.

  3. Initializing a Repository

    A Git repository is initialized to start tracking files in a directory. This creates a hidden `.git` folder to store version history. 

    git init
  4. Staging and Committing

    Staging prepares files to be committed, and committing saves a snapshot of those changes to the repositoryâ€™s history. 

    git add .  # Stage all changes
git commit -m "Initial commit"  # Commit with a message
  5. Branching

    Branching allows you to create separate lines of development within the same repository. The default branch is typically `main`. 

    git branch feature  # Create a new branch
git checkout feature  # Switch to the branch
# Or combine: git checkout -b feature
  6. Merging

    Merging integrates changes from one branch into another, combining their histories. 

    git checkout main  # Switch to main branch
git merge feature  # Merge feature branch into main
  7. Remote Repositories

    Remote repositories (e.g., on GitHub) allow you to store and share your Git repository online. Common commands include pushing and pulling changes. 

    git remote add origin https://github.com/user/repo.git  # Link to remote
git push -u origin main  # Push to remote
git pull origin main  # Pull from remote
  8. Pull Requests

    A GitHub feature that allows you to propose changes from one branch to another (e.g., from a feature branch to `main`). It facilitates code review and collaboration.

    (Performed via GitHub UI, not a Git command)

  9. Rebasing

    Rebasing rewrites commit history by moving or combining a branch onto another, creating a linear history as an alternative to merging. 

    git checkout feature  # Switch to feature branch
git rebase main  # Rebase feature onto main
  10. Git Stash

    Temporarily saves uncommitted changes so you can switch branches or perform other tasks, then reapply them later. 

    git stash  # Save changes
git stash pop  # Reapply stashed changes
  1. Create and Push a Repository

    Initialize a Git repository, add a file, commit it, and push it to GitHub. 

    git init
echo "Hello World" > README.md
git add README.md
git commit -m "Add README"
git remote add origin https://github.com/user/repo.git
git push -u origin main
    Output: Repository created and pushed to GitHub
