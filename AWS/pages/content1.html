Cloud Computing

Overview

  • Typical Flow of Data: User <--> Internet <--> Server
  • Server is Composed of (Aspect of Compute, Memory, Storage, Network):
    1. CPU, Ram
    2. Storage: Data
    3. Database: Store Data in a Structured Way
    4. Network: Routers, Switch, DNS Server

IT Terminologies

  • Network: Cables, Routers, and Servers contected to each other.
  • Router: A Networking device that forwards data packets between computer networks.
  • Switch: Takes a packet and sends it to the correct server/client on the network.

Problem with Traditional IT

  • Pay for the rent for the data center.
  • Pay for Power Supply, COoling , Maintenance
  • Adding and replacing hardware takes time.
  • Limited Scaling.
  • Requires 24/7 for monitoring the infrastructure.
  • How to deal with Disaster?

What is Cloud Computing

  • Cloud Computing is the on-demand delivery of compute power, database storage, aplicatins, and other IT resources
  • Through a cloud services platform with pay-as-you-go pricing
  • YOu can access as many resources as you need, almost instantly
  • Simple way to access servers, storage, databases and a set of application services over the internet.

Types of Cloud Computing

  • IAAS:
    • Provide building blocks for cloud IT
    • Provide networking, computers, data storage space
    • Highest level of flexibility
    • Easy parallel with traditional on-premises IT
    • EG: Amazon EC2, GCP, Azure, Rackspace, Digital Ocean, Linode
  • PAAS:
    • Removes the need for you organization to manage the underlying infrastructure
    • Focus on the deployment and management of your applications
    • Elastic Beanstalk(AWS)
    • EG: Heroku, Google App Engine(GCP), Windows Azure(Microsoft)
  • SAAS:
    • Completed producted that is run and managed by the service provider.
    • EG: Rekognition for ML(AWS), Gmail, Dropbox, Zoom

Pricing of the CLoud - Quick Overview

AWS has 3 pricing fundamentals, following the pay-as-you-go pricing model

  • Compute: Oay for the compute time.
  • Storage: Pay for data stored in the Cloud.
  • Networking: Data Transfer Out of the Cloud. Data transfer IN is free.

Common Questions!!

  1. AWS Regions
    • AWS Region is a cluster of Data Centers
    • Most AWS services are region-scoped, while few are Global-scoped.
  2. Factors for choosing AWS Region
    • Compliance (Data Governance and Legal Requirements)
    • Proximity to customers
    • Available Service
    • Pricing
  3. AWS Availability Zones
    • Each region has minium 3 and maximum 6 availability zones.
    • availability zones are discrete data centers, i.e.,
      • Isolated from disasters
      • Connected with high bandwidth, ultra-low latency networking.
